In place FFT using MPI parallelization. Every process will compute a sub tree of the iterational Cooley-tuekey algorithm, until a dependency between them will occure in the butterfly step. Then, only one process will complete the remaining steps of the algorithm, precisely doing log(p_size) steps with p_size being the number of available processes.

Remarks:

In this implementation of the iterative algorithm, there are two main phases:

P1.) The bit reversal permutation of the vector elements P2.) The "butterfly" operations

MPI is thought to express coarser parallelism, distributing workload on multiple CPUs, by adopting the Message passing memory model.

This makes it a good choice for parallelizing tasks on clusters or distributed memory systems where the data is too large to fit into a single node's memory.

Parallelizing P2 is easy, because the processes will work indipendently on different chunks of the input.

Parallelizing P1, with this model, leads to a tradeoff between space complexity and time complexity.

1.) The first approach would be equally dividing the input on different chunks for each process and run several message passings between processes to retrieve the data to swap, when the process doesn't own it. 2.) Every process would contain a copy of the global data, and there would be no need to comunicate.

The second approach has been implemented, being the library time-optimization oriented.

The1-D vector to transorm, already loaded by each of the processes.
